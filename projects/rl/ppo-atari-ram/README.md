# PPO Atari RAM

Proximal Policy Optimization (PPO) with Generalized Advantage Estimation (GAE) on Atari 2600 games using RAM observations (128-dim vectors). Supports 63 ALE environments via a two-tier config system.

## Usage

```bash
# Local training (default envs: Pong, Asterix, MsPacman)
python train.py [--envs ALE/Pong-v5 ALE/Asterix-v5 ALE/MsPacman-v5]

# Colab deployment
cb launch.py [--args]
```

## Hyperparameters (defaults)

| Parameter | Value |
|-----------|-------|
| Total timesteps | 500,000 (per env) |
| Num vectorized envs | 4 |
| Num steps per rollout | 128 |
| Learning rate | 2.5e-4 |
| Gamma | 0.99 |
| GAE lambda | 0.95 |
| Clip coefficient | 0.1 |
| Entropy coefficient | 0.01 |
| Value function coefficient | 0.5 |
| Max gradient norm | 0.5 |
| PPO epochs | 4 |
| Num minibatches | 4 |

## Network architectures

All networks take a 128-dim RAM observation (uint8, scaled to [0,1]) and output discrete action logits + scalar value.

| Key | Architecture |
|-----|-------------|
| `mlp-small` | [128, 128] hidden layers |
| `mlp-medium` | [256, 256] hidden layers |
| `mlp-large` | [512, 512] hidden layers |
| `resmlp` | [256, 256] with LayerNorm and residual skip connections |

All layers use orthogonal initialization; actor head uses gain=0.01.

## Environments

63 Atari games supported. Configs define per-env: `n_actions`, `solved_threshold`, `network`, and `total_timesteps`. Generated by `generate_configs.py`.

## Gotchas

- Observations are `obs_type="ram"` (128-byte RAM state), not screen pixels.
- Asynchronous vectorized environments via `gym.vector.AsyncVectorEnv` with `num_envs=4`.
- Config filenames use hyphens (e.g., `ALE-Pong-v5.json` for env ID `ALE/Pong-v5`).
- Advantages are normalized across the full batch before the PPO update.
