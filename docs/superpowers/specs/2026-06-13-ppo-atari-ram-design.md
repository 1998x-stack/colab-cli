# PPO on Atari RAM — Design Spec

**Date:** 2026-06-13
**Goal:** Pluggable PPO agent for Atari RAM environments, deployable on Colab, with cron-based monitoring.

## Architecture

CleanRL-style training loop with pluggable Network / Env / Algorithm components. Each component is its own file with a uniform interface.

### File structure

```
projects/ppo-atari-ram/
├── configs/                    # 63 JSON files, one per ALE/*-ram-v5
│   ├── _defaults.json          # shared PPO hyperparams
│   ├── ALE-Pong-ram-v5.json    # per-env overrides (n_actions, solved_threshold, ...)
│   └── ... (auto-generated)
├── train.py                    # argparse, training loop, metrics, plotting
├── ppo_agent.py                # PPOAgent: act(), compute_returns(), update()
├── networks.py                 # MLPActorCritic, registry (mlp-small/medium/large, resmlp)
├── env_factory.py              # make_ram_env(): creates ALE RAM env + wrappers
├── config_loader.py            # loads _defaults + per-env JSON merge
├── generate_configs.py         # auto-generate 63 env configs from ALE registry
├── launch.py                   # Colab detached launcher
├── fetch.sh                    # cron: tar → download → extract → report
└── output/                     # synced from VM
```

### Component interfaces

**Network (networks.py):** Pure PyTorch. No RL knowledge.
```python
class MLPActorCritic(nn.Module):
    def __init__(self, obs_dim=128, n_actions=4, hidden=[256, 256]): ...
    def forward(self, x) -> (action_logits, value): ...
    def get_action_and_value(self, x, action=None) -> (action, log_prob, entropy, value): ...
    def init_weights(self): ...  # orthogonal init
```

**Agent (ppo_agent.py):** Receives a network. No env knowledge.
```python
class PPOAgent:
    def __init__(self, network, lr, gamma, gae_lambda, clip_coef, ent_coef, 
                 vf_coef, max_grad_norm, n_epochs, n_minibatches, device): ...
    def act(self, obs) -> (action, log_prob, value): ...
    def compute_returns(self, next_obs, next_done, rewards, dones, values) -> (advantages, returns): ...
    def update(self, rollouts) -> dict: ...  # returns {"pg_loss": ..., "vf_loss": ..., ...}
```

**Env (env_factory.py):** Pure Gymnasium. No agent knowledge.
```python
def make_ram_env(env_id: str, num_envs: int = 4) -> gym.vector.AsyncVectorEnv: ...
```

**Config (config_loader.py):** Merges tiered configs.
```python
def load_config(env_id: str, config_dir: str = "configs") -> dict: ...
```

## Network architecture

Shared trunk MLP for actor-critic. RAM input `(128,)` uint8 → `/255.0` → `(128,)` float.

| Registry key | Hidden dims | Params | Use case |
|---|---|---|---|
| `mlp-small` | [128, 128] | ~40K | Simple games (Pong) |
| `mlp-medium` | [256, 256] | ~120K | Default, most games |
| `mlp-large` | [512, 512] | ~350K | Complex games (MsPacman) |
| `resmlp` | [256, 256] + skip | ~200K | Deeper, residual |

All use: shared trunk → actor head (Linear → n_actions) + critic head (Linear → 1).
Activation: ReLU. Init: orthogonal with sqrt(2) gain.

## Config system

Two-tier: `_defaults.json` (shared) → per-env JSON (overrides).

Per-env JSON overrides at minimum: `env_id`, `n_actions`, `solved_threshold`.
Optional overrides: `network`, `total_timesteps`, any hyperparam.

All 63 `ALE/*-ram-v5` envs get a config generated by `generate_configs.py`.
Only 3 actually run: Pong (easy), Asterix (medium), MsPacman (hard).

## Training loop (CleanRL pattern)

```
For env in [Pong, Asterix, MsPacman]:
  1. Create N=4 parallel envs via AsyncVectorEnv
  2. Create network (from config) + PPOAgent
  3. For iteration in 1..(total_steps / (num_envs * num_steps)):
     a. Collect rollouts: for t in 1..num_steps: act, step, store
     b. Compute GAE advantages + returns
     c. PPO update × K=4 epochs, minibatch shuffle
     d. Every eval_interval: eval on 5 episodes, greedy policy
     e. Every plot_interval: overwrite PNGs
     f. Log to file + CSV
  4. Save best checkpoint, final metrics
```

Step budget: num_envs=4, num_steps=128, so 128×4=512 steps per iteration.
500K total → ~977 iterations per env. At ~1000 steps/sec (MLP is fast), ~8 min per env.

## Outputs

```
/content/ppo-atari-output/
├── logs/train.log           # timestamped per-iteration
├── pngs/
│   ├── training_curves.png  # reward + loss + entropy + clip_frac
│   └── comparison.png       # all trained envs overlaid
├── metrics.csv              # iteration,env,steps,reward,loss,entropy,...
└── checkpoints/<env>_best.pt
```

Log format: `[HH:MM:SS] iter N/M | Env: reward=X avg10=Y | vf_loss=A pg_loss=B | ent=C clip=D | steps=E`

CSV columns: `iteration,env,total_steps,mean_reward,avg10_reward,eval_reward,pg_loss,vf_loss,entropy,clip_frac,lr,elapsed_s`

PNG panels (2×2): reward curve (raw + avg10 + solved line), policy+value losses, entropy, clip fraction.

## Deployment

1. Provision: `colab new --gpu T4 -s ppo-atari`
2. Upload: train.py, ppo_agent.py, networks.py, env_factory.py, config_loader.py, configs/*.json
3. Launch: `LAUNCH_SCRIPT="train.py" LAUNCH_DEPS="gymnasium[atari],ale-py,matplotlib" colab exec -f launch.py`
4. Cron: every 2 min → tar output on VM → download → extract → report tail + CSV + PNGs

## Success criteria

- [ ] All 63 env configs generated with correct n_actions
- [ ] PPO trains on Pong, Asterix, MsPacman without crashing
- [ ] Reward improves above random baseline on all 3 environments
- [ ] PNGs, CSV, logs generated and fetchable via cron
- [ ] Cron watchtower reports training health every 2 minutes
- [ ] No hardcoded hyperparams — everything driven by config JSON
